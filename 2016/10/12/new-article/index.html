修车铺也终于开业了

空闲时间也算是利用起来了，在修车。。呃不对，是在编码这条路上也算越走越稳。。呃呸，是越走越顺的情况下，终于开通了梦寐以求的博客了，感谢我的好友军哥的帮助与指导。

好了进入正题吧，使用Hexo搭建一个还算顺眼的博客吧。。。。

安装必要的软件

这里软件均为windows版，其他的不清楚啦啦啦(・ω・=),本人系统是Win10 64位专业版

Git

经常使用的Git以及Github，首先安装Git：

  • 下载Git

选择合适的版本下载然后安装，然后。。。。。。。

放着 (:3[▓▓] (:3[▓▓▓▓▓▓▓▓▓] (¦3[▓▓]

Node.js

早就耳闻Node的大名，没想到竟然在这里用上了

这个主要用来渲染博客静态页面用，速度挺快

  • 下载Node.js

同样选择电脑操作系统对应版本

Hexo

先打开Git Bash,输入下面的信息进行安装Hexo

npm install hexo-cli -g

安装成功后，输入下面的命令来查看Hexo的版本信息：

hexo -v

或者使用下面的不带-的命令，都可以查看

hexo v

出现了下面的类似信息就说明安装成功

$ hexo -v

MarkdownPad2

用来编写博客用的工具，语法使用也很方便下面破解版地址

  • 下载MarkdownPad2破解版

安装完成后，使用压缩包里面的exe替换掉安装路径下的同名文件

然后能满怀欣喜的破解完，打开后发现提示错误，不停的弹出

markdownpad2 html渲染组件出错，原来win10下有这个问题，解决办法下载下面的HTML UI ENGINE安装即可

GitHub客户端

这个客户端工具看起来比较舒服（主要有我喜欢的深色主题），挺方便的一个工具

  • 下载GitHub客户端 　 　

    注册Github账号可以看Github的注册与使用（详细图解）
    　　
    进一步想深入学习Github的可以看一下这个博客 从0开始学习Github系列汇总

登录你的Github账号，创建一个Repository（仓库），仓库的名字必须是 你的github的用户名.github.io （这是固定写法）

在这里假设你的用户名是zhangsan，则你要创建的仓库名就是zhangsan.github.io

用 终端 或者 Github Desktop软件 Clone zhangsan.github.io 仓库到本地你指定的目录下。

Hexo的使用

好了，现在就可以愉快的修车，咳咳咳，使用博客了

创建博客目录

尽量创建的路径不要包含中文字符与空格，假设创建的路径在D盘，创建一个名叫blog的文件夹，然后右键使用Git Bash打开输入命令：

hexo init

初始化博客，然后执行

npm install

创建好之后，可以看到里面的目录，大概是如下结构：

.
├── _config.yml
├── node_modules
├── package.json
├── scaffolds
├── source
|   └── _posts
└── themes

这时候，基本可以写一篇文章测试一下了

写文章

同样还是选中blog文件夹，右键Git Bash打开，执行如下：

hexo new "New_Word"

找到blog里面的New_Word.md,使用MarkdownPad2打开

---
title: windows下用Hexo创建个人博客
date: 2016-10-12 18:19:30
tags:
---

这个就是最初的内容了，在下方写点内容

# Hello Word
(。・ˍ・。)一本正经

保存后就可以进行生成静态页面了

hexo g

页面预览

好了，同样右键选中blog，点击Git Bash,然后输入如下命令：

hexo s

或者完整命令：

hexo server

弹出如下信息：

INFO  Hexo is running at http://0.0.0.0:4000/. Press Ctrl+C to stop.

这样服务就发布完成了（当然仅仅只是本地）

打开浏览器输入http://localhost:4000/，就能看到本地发布的博客了！

部署

我们创建好博客之后，要怎么放到互联网上给世界友人访问到呢？

在部署之前呢，找到你的本地GitHub的路径，默认是在C盘``文档中，进去后可以看到类似411805576.github.io这样的文件夹，建议先删除掉里面的所有内容，打开GitHub客户端工具，进行提交，然后再把blog中的public文件中的所有内容丢进去，再次提交。ok后，回到hexo，使用如下命令进行部署：

hexo d

或者完整命令：

hexo deploy

其中有提示警告，忽略即可，部署成功后，访问你的部署路径类似https://411805576.github.io/，411805576为我的用户名，换成自己的用户名即可。

最后呢，老是觉得这个主题太千篇一律了，想换主题怎么办呢？

主题使用

这个主题呢，度娘一抓一大把，这里使用一个例子Apollo来说明吧

  • 先执行下面的命令进行安装主题：

    hexo init Blog
cd Blog 
npm install
npm install --save hexo-renderer-jade hexo-generator-feed hexo-generator-sitemap hexo-browsersync hexo-generator-archive
git clone https://github.com/pinggod/hexo-theme-apollo.git themes/apollo

  • 修改blog下面_config.yml的theme为apollo

    theme: apollo

    archive_generator:

    per_page: 0

    yearly: false

    monthly: false

    daily: false

  • 更新

    cd themes/apollo

    git pull
